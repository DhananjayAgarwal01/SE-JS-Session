JavaScript Basics

Theory (simple)

  • Variable: a named box that stores a value
  • Types: number, text (string), true/false (boolean)
  • Expression: a small calculation like 10 + 2
  • Output: console.log writes to the console; alert shows a popup
// Say something on the screen
console.log("Hello!"); // shows in the console
alert("Hi there!"); // small popup

// Variables: store a value let name = "Sam"; // text (string) let age = 18; // number let isStudent = true; // true/false (boolean)

// Do some math let a = 10; let b = 2; let sum = a + b; // 12 console.log("Sum:", sum);

üíª Try: Change the values and see what happens.

Arrays and Loops

Theory

  • Array: an ordered list of values (first item is at index 0)
  • Loop: repeat the same steps for each item in the list
  • Accumulator: a variable that keeps a running total
// A list of numbers (array) let scores = [10, 20, 30, 40];

// Go through each score with a loop let total = 0; for (let i = 0; i < scores.length; i++) { total = total + scores[i]; } let average = total / scores.length; console.log("Average:", average);

Make the Page React (DOM)

Theory

  • DOM: the page represented as a tree of elements
  • Select: get elements by id (getElementById) or class (querySelector)
  • Change: update textContent or style to change what you see
  • Events: run code when the user clicks or types
// Get an element from the page const box = document.getElementById('results');

// Change the text inside it box.textContent = 'Hello from JavaScript!';

// React to a button click const btn = document.querySelector('.say-hi'); btn.addEventListener('click', function () { box.textContent = 'You clicked the button!'; });

üåê What is JSON (Simple Data)?

Theory

  • JSON: a simple text format to store/share data
  • Objects: { key: value } pairs; Arrays: [ item, item ]
  • Rule: in JSON text, keys and strings use double quotes
  • Use: JSON.stringify (object ‚Üí text), JSON.parse (text ‚Üí object)
// JSON looks like objects and arrays let student = { name: "Asha", age: 19 }; let group = ["Asha", "Manu", "Zara"];

// Turn object -> text, and back let asText = JSON.stringify(student); console.log(asText); let backToObject = JSON.parse(asText); console.log(backToObject.name);

üåâ Getting Ready for React

  • React: A library to build user interfaces
  • Idea: UI = function of state (data)
  • Small pieces: Components
  • Data in: Props; data that changes: State

Theory: When state changes, React re-renders the UI automatically.

// Plain JS thinking function greet(name) { return "Hello, " + name; } console.log(greet("Asha")); // React thinking (conceptual) // Component = function that returns UI // State = data that can change

üß© JSX: HTML inside JavaScript

JSX looks like HTML, but it's JavaScript syntax. Browsers compile it.

  • Looks like HTML but it is JavaScript
  • Curly braces {} show values from variables
  • One parent element must wrap the returned JSX
// JSX example (we'll do this in the React session) function Hello(props) { return ( <div> <h3>Hello, {props.name}</h3> </div> ); }

Key rule: In JSX, use {} to show a JavaScript value.

üß± Components, Props, and State

Concepts

  • Component: A reusable UI piece
  • Props: Inputs to a component
  • State: Data that can change over time
  • Event: A user action (click, type)

Preview Code

// A tiny React component (preview) function Counter() { const [count, setCount] = React.useState(0); return ( <div> <p>Count: {count}</p> <button onClick={() => setCount(count + 1)>+1</button> </div> ); }

Mental model: Change state ‚Üí React re-renders the UI.
